The logical/xnor.h header defines a logical XNOR operator.

Usage

#include <chaos/preprocessor/logical/xnor.h>

Contents

  • CHAOS_PP_XNOR

CHAOS_PP_XNOR

The CHAOS_PP_XNOR macro expands to the logical XNOR of its operands.

Usage 

CHAOS_PP_XNOR(p)(q)
p

The left operand. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

q

The right operand. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

Remarks

If either p or q is non-zero exclusively, this macro expands to 0. Otherwise, it expands to 1.

This macro performs a Boolean conversion on each operand. If this conversion is undesirable, CHAOS_PP_BITXNOR may be used instead.

See Also

  • CHAOS_PP_BITXNOR

Sample Code

#include <chaos/preprocessor/logical/xnor.h>

CHAOS_PP_XNOR(4)(7) // 1
CHAOS_PP_XNOR(5)(0) // 0

CHAOS_PP_XNOR_ID

The CHAOS_PP_XNOR_ID macro indirectly expands to CHAOS_PP_XNOR.

Usage 

CHAOS_PP_XNOR_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_XNOR_

The CHAOS_PP_XNOR_ macro expands to a λ expression representing CHAOS_PP_XNOR.

Usage - C99 Specific 

CHAOS_PP_XNOR_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_XNOR)
