The punctuation/hash.h header defines macros that expand to hash marks.

CHAOS_PP_HASH

The CHAOS_PP_HASH macro expands to a single hash mark.

Usage 

CHAOS_PP_HASH()

Remarks

The hash mark preprocessing token (#) may not be used in the replacement list of a function-like macro unless it is used as the stringizing operator. This macro facilitates the introduction of the hash mark preprocessing token in such contexts.

See Also

  • CHAOS_PP_DIGRAPH_HASH

Sample Code

#include <chaos/preprocessor/punctuation/hash.h>
#include <chaos/preprocessor/stringize.h>

#define FUNCTION(p) \
    CHAOS_PP_STRINGIZE(CHAOS_PP_HASH() p) \
    /**/

FUNCTION(abc) // "# abc"

CHAOS_PP_HASH_ID

The CHAOS_PP_HASH_ID macro indirectly expands to CHAOS_PP_HASH.

Usage 

CHAOS_PP_HASH_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_HASH_

The CHAOS_PP_HASH_ macro expands to a λ expression representing CHAOS_PP_HASH.

Usage - C99 Specific 

CHAOS_PP_HASH_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_HASH)

Usage 

CHAOS_PP_HASH_HASH()

Remarks

The double hash mark preprocessing token (##) may not be used in the replacement list of a macro unless it is used as the token-pasting operator. This macro facilitates the introduction of the double hash mark preprocessing token in such contexts.

See Also

  • CHAOS_PP_DIGRAPH_HASH_HASH

#include <chaos/preprocessor/punctuation/hash.h>

    CHAOS_PP_STRINGIZE(p CHAOS_PP_HASH_HASH() p) \
    /**/

FUNCTION(x) // "x ## x"
