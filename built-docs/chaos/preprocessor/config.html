The config.h header defines library configuration macros.

Usage

#include <chaos/preprocessor/config.h>

Contents

CHAOS_PP_VARIADICS

The CHAOS_PP_VARIADICS macro determines whether the library supports C99/C++0x facilities such as variadic macros and placemarkers.

Usage 

#define CHAOS_PP_VARIADICS x
[x]

The value that determines whether variadics and placemarkers are supported.

Usage 

CHAOS_PP_VARIADICS

Remarks

This macro defaults to 1 if the preprocessor is a C99 preprocessor. Otherwise, it defaults to 0.

This macro may be defined by the user prior to the inclusion of any library headers to explicitly enable or disable the use of variadics and placemarkers in the library. If this macro is defined by the user, the library redefines it such that it expands to either 0 or 1. If it is defined by the user with an empty replacement list or is defined as non-zero, the library redefines it as 1. Otherwise, the library redefines it as 0.

If the user defines this macro as an expression (as opposed to a number), the expression should be enclosed in parentheses to avoid interference with the redefinition process. 

#define CHAOS_PP_VARIADICS (A + B)

This macro may further be used in user code to distinguish between variadic macro and placemarker support and the lack thereof.

Sample Code

#include <chaos/preprocessor/config.h>

#if CHAOS_PP_VARIADICS
    // ...
#else
    // ...
#endif

CHAOS_PP_VARIADICS_ID

The CHAOS_PP_VARIADICS_ID macro indirectly expands to CHAOS_PP_VARIADICS.

Usage 

CHAOS_PP_VARIADICS_ID()

Remarks

This macro effectively allows CHAOS_PP_VARIADICS to be passed around without expansion.

CHAOS_PP_VARIADICS_

The CHAOS_PP_VARIADICS_ macro expands to a λ expression representing CHAOS_PP_VARIADICS.

Usage - C99 Specific 

CHAOS_PP_VARIADICS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_VARIADICS_ID)()
