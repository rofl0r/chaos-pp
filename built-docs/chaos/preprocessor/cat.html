The cat.h header defines basic concatenation macros.

Usage

#include <chaos/preprocessor/cat.h>

Contents

  • CHAOS_PP_CAT
  • CHAOS_PP_PRIMITIVE_CAT

CHAOS_PP_CAT

The CHAOS_PP_CAT macro concatenates its arguments after they have been expanded.

Usage - C99 Specific 

CHAOS_PP_CAT(a, ...)
[a]

The left operand of the concatenation.

[...]

The right operand of the concatenation.

Usage - C/C++ Specific 

CHAOS_PP_CAT(a, b)
a

The left operand of the concatenation.

b

The right operand of the concatenation.

Remarks

This macro concatenates the rightmost preprocessing token of a to the leftmost preprocessing token of b or __VA_ARGS__. 

a ## __VA_ARGS__ // C99
a ## b           // C and C++

This macro allows its arguments to expand on input. This macro's sibling, CHAOS_PP_PRIMITIVE_CAT, does not provide this delay.

In C99, if both a and __VA_ARGS__ expand to placemarkers, this macro expands to nothing. 

CHAOS_PP_CAT(,) //

Similarly, if either a or __VA_ARGS__ expands to a placemarker exclusively, this macro expands to the argument that does not expand to a placemarker. 

CHAOS_PP_CAT(123,)   // 123
CHAOS_PP_CAT(, a, b) // a, b

Sample Code

#define A 1
#define B 2

CHAOS_PP_CAT(A, B) // 12

CHAOS_PP_CAT_ID

The CHAOS_PP_CAT_ID macro indirectly expands to CHAOS_PP_CAT.

Usage 

CHAOS_PP_CAT_ID()

Remarks

This macro effectively allows CHAOS_PP_CAT to pass through itself without becoming disabled.

CHAOS_PP_CAT_

The CHAOS_PP_CAT_ macro expands to a λ expression representing CHAOS_PP_CAT.

Usage - C99 Specific 

CHAOS_PP_CAT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_CAT_ID)()

CHAOS_PP_PRIMITIVE_CAT

The CHAOS_PP_PRIMITIVE_CAT macro concatenates its arguments before they are expanded.

Usage - C99 Specific 

CHAOS_PP_PRIMITIVE_CAT(x, ...)
[a]

The left operand of the concatenation.

[b]

The right operand of the concatenation.

Usage - C/C++ Specific 

CHAOS_PP_PRIMITIVE_CAT(a, b)
a

The left operand of the concatenation.

b

The right operand of the concatenation.

Remarks

This macro behaves identically to CHAOS_PP_CAT except that it does not allow its arguments to expand prior to concatenation.
