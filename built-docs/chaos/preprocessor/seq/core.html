CHAOS_PP_SEQ

The CHAOS_PP_SEQ macro designates the name of the sequence data structure.

Usage 

CHAOS_PP_SEQ

Remarks

This macro expands to CHAOS_PP_SEQ. In other words, the identifier CHAOS_PP_SEQ is defined as itself.

CHAOS_PP_SEQ_NIL

The CHAOS_PP_SEQ_NIL macro expands to a nil sequence.

Usage 

CHAOS_PP_SEQ_NIL()

Remarks

In C99, this macro expands to nothing. (More generally, a nil sequence is an empty sequence of preprocessing tokens.) In C++ (and C prior to C99), there is no such thing as a nil sequence because "nothing" cannot be passed as an argument. Hence, in C++ (and C prior to C99), this macro expands to a rogue value (that is uniquely recognized by the other macros in this header) such that generic algorithms can behave as if nil sequences exist.

See Also

  • CHAOS_PP_ARRAY_NIL
  • CHAOS_PP_LIST_NIL
  • CHAOS_PP_NIL
  • CHAOS_PP_STRING_NIL
  • CHAOS_PP_TUPLE_NIL
  • CHAOS_PP_VARIADIC_SEQ_NIL

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/seq/core.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_SEQ_NIL() //

#else

CHAOS_PP_SEQ_NIL() // unspecified

#endif

CHAOS_PP_SEQ_CONS(
    CHAOS_PP_SEQ_CONS(
        CHAOS_PP_SEQ_NIL(),
        b
    ),
    a
)
// (a)(b)

CHAOS_PP_SEQ_CONS

The CHAOS_PP_SEQ_CONS macro appends an element to the head of a sequence.

Usage - C99 Specific 

CHAOS_PP_SEQ_CONS(seq, ...)
[seq]

The sequence to which __VA_ARGS__ is to be appended.

[...]

The element to be appended to the head of seq.

Usage - C/C++ Specific 

CHAOS_PP_SEQ_CONS(seq, x)
seq

The sequence to which x is to be appended.

x

The element to be appended to the head of seq.

Remarks

This macro expands to a sequence containing the elements of seq with x (or __VA_ARGS__) appended to the head.

See Also

  • CHAOS_PP_ARRAY_CONS
  • CHAOS_PP_CONS
  • CHAOS_PP_LIST_CONS
  • CHAOS_PP_STRING_CONS
  • CHAOS_PP_TUPLE_CONS
  • CHAOS_PP_VARIADIC_SEQ_CONS

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/seq/core.h>

CHAOS_PP_SEQ_CONS((y)(z), x) // (x)(y)(z)

#if CHAOS_PP_VARIADICS

CHAOS_PP_SEQ_CONS((a, b)(c, d), +, -) // (+, -)(a, b)(c, d)
CHAOS_PP_SEQ_CONS(, x)                // (x)

#else

CHAOS_PP_SEQ_CONS(CHAOS_PP_SEQ_NIL(), *) // (*)

#endif

CHAOS_PP_SEQ_ITEM

The CHAOS_PP_SEQ_ITEM macro is an identity macro used to encapsulate elements.

Usage - C99 Specific 

CHAOS_PP_SEQ_ITEM(...)
[...]

The element of a sequence to be encapsulated.

Usage - C/C++ Specific 

CHAOS_PP_SEQ_ITEM(x)
x

The element of a sequence to be encapsulated.

Remarks

This macro is used by generic algorithms to encapsulate elements before they are passed to user-defined macros.

Though elements of a sequence can have more than one sub-element in C99, this macro assumes that each element is the same arity. As such, it is a simple identity macro. (The CHAOS_PP_VARIADIC_SEQ_ITEM macro, on the other hand, assumes that elements have differing arities.)
