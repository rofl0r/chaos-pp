The repetition/parametric/for_aux.h header defines a predicate repetition algorithm which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/parametric/for_aux.h>

Contents

CHAOS_PP_FOR_AUX_PARAMETRIC

The CHAOS_PP_FOR_AUX_PARAMETRIC macro is a predicated repetition construct.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_AUX_PARAMETRIC(size, pred, op, macro, data, ...))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

pred

The predicate that determines whether the algorithm should continue. This predicate must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

op

The operation used to iterate the state. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

macro

The target to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, data†, state‡)

† If the auxiliary data is parenthetic, the data pseudo-parameter must be the number of parameters that corresponds to the number of sub-elements in the auxiliary data. Otherwise, it must be a single parameter.

‡ The state pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the initial state, __VA_ARGS__.

[data]

Auxiliary data to be passed through the algorithm to the predicate, operation, and target. This argument may be either a parenthetic expression of any arity or a non-parenthetic expression.

[...]

The initial state.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_AUX_PARAMETRIC(size, pred, op, macro, data, state))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

pred

The predicate that determines whether the algorithm should continue. This predicate must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

pred(s, data, state)
op

The operation used to iterate the state. This operation must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, data, state)
macro

The target to be repeated. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, data, state)
[data]

Auxiliary data to be passed through the algorithm to the predicate, operation, and target. This argument may be either a unary parenthetic expression or a non-parenthetic expression.

state

The initial state.

Remarks

This macro behaves identically to CHAOS_PP_FOR_AUX except that the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the predicate and the operation have at least size recursion steps available whenever they are invoked.

Ignoring the complexity of pred, op, and macro (each of which is invoked once for each variation of the state), each iteration of the state requires only a single recursion step. This implies that the algorithm is capable of repeating the target p∙(sω-s-1) times, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, and s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()).

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_AUX_PARAMETRIC_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_AUX_PARAMETRIC_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FOR_AUX_PARAMETRIC_ID

The CHAOS_PP_FOR_AUX_PARAMETRIC_ID macro indirectly expands to CHAOS_PP_FOR_AUX_PARAMETRIC.

Usage 

CHAOS_PP_FOR_AUX_PARAMETRIC_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR_AUX_PARAMETRIC to pass through itself without becoming disabled.

CHAOS_PP_FOR_AUX_PARAMETRIC_

The CHAOS_PP_FOR_AUX_PARAMETRIC_ macro expands to a λ expression representing CHAOS_PP_FOR_AUX_PARAMETRIC.

Usage - C99 Specific 

CHAOS_PP_FOR_AUX_PARAMETRIC_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_FOR_AUX_PARAMETRIC_ID)()
