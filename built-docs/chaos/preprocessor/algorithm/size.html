The algorithm/size.h header defines an algorithm that counts the elements in a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/size.h>

Contents

  • CHAOS_PP_SIZE

CHAOS_PP_SIZE

The CHAOS_PP_SIZE macro counts the elements in its argument.

Usage 

CHAOS_PP_SIZE(g)
g

The generic data structure whose elements are to be counted.

Remarks

This macro expands to the number of elements in g. If g contains more than CHAOS_PP_LIMIT_MAG elements, the result is saturated to CHAOS_PP_LIMIT_MAG.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_SIZE_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/size.h>
#include <chaos/preprocessor/string/core.h>

CHAOS_PP_SIZE((CHAOS_PP_STRING) C 9 9) // 3

CHAOS_PP_SIZE_ID

The CHAOS_PP_SIZE_ID macro indirectly expands to CHAOS_PP_SIZE.

Usage 

CHAOS_PP_SIZE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_SIZE_

The CHAOS_PP_SIZE_ macro expands to a λ expression representing CHAOS_PP_SIZE.

Usage - C99 Specific 

CHAOS_PP_SIZE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SIZE)

CHAOS_PP_SIZE_BYPASS

The CHAOS_PP_SIZE_BYPASS macro behaves identically to CHAOS_PP_SIZE except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_SIZE_BYPASS(s, g)
s

The current bypass recursion state.

g

The generic data structure whose elements are to be counted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_SIZE.
