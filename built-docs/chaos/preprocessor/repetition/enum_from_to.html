The repetition/enum_from_to.h header defines an algorithm that enumerates a user-specified target for each number in a specified range.

Usage

#include <chaos/preprocessor/repetition/enum_from_to.h>

Contents

CHAOS_PP_ENUM_FROM_TO

The CHAOS_PP_ENUM_FROM_TO macro enumerates its third argument for each number in the range specified by its first two arguments.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_FROM_TO(from, to, macro))
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

Remarks

The CHAOS_PP_ENUM_FROM_TO macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of enumerating the target with a comma for each number ranging from from to to - 1. This is illustrated by the following pseudo-code: 

macro(s, from, data), macro(s, from + 1, data), ... macro(s, to - 1, data)

The target is invoked for each number in the half-open range [from, to) with the current recursion state (s), the current repetition (n), and, if it exists, the auxiliary data (data or __VA_ARGS__). This target may expand to a deferred expression in terms of CHAOS_PP_NEXT(s). This allows the target to recurse on itself if necessary.

In C99, it is possible to omit the auxiliary data argument altogether. This is different than passing "nothing" as the auxiliary data. 

CHAOS_PP_EXPR(
    CHAOS_PP_ENUM_FROM_TO(0, 10, SEP, MACRO,) // #1
    CHAOS_PP_ENUM_FROM_TO(0, 10, SEP, MACRO)  // #2
)

The first line passes "nothing" as the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0,)

The second line omits the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0)

The fundamental difference is that, in the first line, the auxiliary data is empty (which might have a valid meaning as data), but, in the second line, there is not auxiliary data. Hence, the algorithm is overloaded on a true-optional parameter.

Invocations of the target are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(s), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

Ignoring the complexity of macro (which is invoked exactly to - from times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(t-f+1), where t is to and f is from.

This macro deduces the current recursion state and invokes CHAOS_PP_ENUM_FROM_TO_S. If the current recursion state is already known, direct use of CHAOS_PP_ENUM_FROM_TO_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/repetition/enum_from_to.h>

#define MACRO(s, n, param) param ## n

CHAOS_PP_EXPR(CHAOS_PP_ENUM_FROM_TO(
    5, 10, MACRO, class T
))
// class T5, class T6, class T7, class T8, class T9

CHAOS_PP_ENUM_FROM_TO_ID

The CHAOS_PP_ENUM_FROM_TO_ID macro indirectly expands to CHAOS_PP_ENUM_FROM_TO.

Usage 

CHAOS_PP_ENUM_FROM_TO_ID()

Remarks

This macro effectively allows CHAOS_PP_ENUM_FROM_TO to pass through itself without becoming disabled.
