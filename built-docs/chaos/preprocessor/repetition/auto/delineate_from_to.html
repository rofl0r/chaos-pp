The repetition/auto/delineate_from_to.h header defines an algorithm that delineates a user-specified target for each number in a specified range.

Usage

#include <chaos/preprocessor/repetition/auto/delineate_from_to.h>

Contents

  • CHAOS_PP_AUTO_DELINEATE_FROM_TO

CHAOS_PP_AUTO_DELINEATE_FROM_TO

The CHAOS_PP_AUTO_DELINEATE_FROM_TO macro delineates its fourth argument for each number in the range specified by its first two arguments.

Usage - C99 Specific 

CHAOS_PP_AUTO_DELINEATE_FROM_TO(from, to, sep, macro)
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific 

CHAOS_PP_AUTO_DELINEATE_FROM_TO(from, to, sep, macro, ...)
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[...]

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_AUTO_DELINEATE_FROM_TO macro is a higher-order thunk that deduces the current higher-order entry point and invokes CHAOS_PP_DELINEATE_FROM_TO.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/facilities/identity.h>
#include <chaos/preprocessor/repetition/auto/delineate_from_to.h>

#if CHAOS_PP_VARIADICS

#define MACRO(s, n) n

CHAOS_PP_AUTO_DELINEATE_FROM_TO(
    5, 10, CHAOS_PP_IDENTITY(+), MACRO
)
// 5 + 6 + 7 + 8 + 9

#endif

CHAOS_PP_AUTO_DELINEATE_FROM_TO_ID

The CHAOS_PP_AUTO_DELINEATE_FROM_TO_ID macro indirectly expands to CHAOS_PP_AUTO_DELINEATE_FROM_TO.

Usage - C99 Specific 

CHAOS_PP_AUTO_DELINEATE_FROM_TO_ID()

Remarks

This macro effectively allows CHAOS_PP_AUTO_DELINEATE_FROM_TO to be passed around without expansion.

CHAOS_PP_AUTO_DELINEATE_FROM_TO_

The CHAOS_PP_AUTO_DELINEATE_FROM_TO_ macro expands to a λ expression representing CHAOS_PP_AUTO_DELINEATE_FROM_TO.

Usage - C99 Specific 

CHAOS_PP_AUTO_DELINEATE_FROM_TO_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_AUTO_DELINEATE_FROM_TO_ID)()
