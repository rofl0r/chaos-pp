The algorithm/auto/transform.h header defines an algorithm that transforms the elements of a generic data structure with a user-specified operation.

Usage

#include <chaos/preprocessor/algorithm/auto/transform.h>

Contents

  • CHAOS_PP_AUTO_TRANSFORM

CHAOS_PP_AUTO_TRANSFORM

The CHAOS_PP_AUTO_TRANSFORM macro transforms each element of its second argument with the operation specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_AUTO_TRANSFORM(op, g)
op

The operation to be used to transform each element of g. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem†)

† If the type of g is non-variadic, elem must be ... or the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be ... or a single parameter.

g

The generic data structure whose elements are to be transformed.

Usage - C99 Specific 

CHAOS_PP_AUTO_TRANSFORM(op, g, ...)
op

The operation to be used to transform each element of g. This operation must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

op(s, elem†, data‡)

† If the type of g is non-variadic, elem must be the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be a single parameter.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

g

The generic data structure whose elements are to be transformed.

[...]

Auxiliary data to be passed through the algorithm to the operation.

Remarks

The CHAOS_PP_AUTO_TRANSFORM macro is a higher-order thunk that deduces the current higher-order entry point and invokes CHAOS_PP_TRANSFORM.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/algorithm/auto/transform.h>
#include <chaos/preprocessor/arithmetic/dec.h>
#include <chaos/preprocessor/array/core.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_AUTO_TRANSFORM(
    CHAOS_PP_DEC_(CHAOS_PP_ARG(1)),
    (CHAOS_PP_ARRAY) (5, (1, 2, 3, 4, 5))
)
// (5, (0, 1, 2, 3, 4))

#endif

CHAOS_PP_AUTO_TRANSFORM_ID

The CHAOS_PP_AUTO_TRANSFORM_ID macro indirectly expands to CHAOS_PP_AUTO_TRANSFORM.

Usage - C99 Specific 

CHAOS_PP_AUTO_TRANSFORM_ID()

Remarks

This macro effectively allows CHAOS_PP_AUTO_TRANSFORM to be passed around without expansion.

CHAOS_PP_AUTO_TRANSFORM_

The CHAOS_PP_AUTO_TRANSFORM_ macro expands to a λ expression representing CHAOS_PP_AUTO_TRANSFORM.

Usage - C99 Specific 

CHAOS_PP_AUTO_TRANSFORM_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_AUTO_TRANSFORM_ID)()
