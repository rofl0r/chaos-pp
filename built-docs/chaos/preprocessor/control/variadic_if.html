The control/variadic_if.h header defines a macro that acts as an "if-else" construct.

Usage

#include <chaos/preprocessor/control/variadic_if.h>

Contents

  • CHAOS_PP_VARIADIC_IF

CHAOS_PP_VARIADIC_IF

The CHAOS_PP_VARIADIC_IF macro expands to one of two alternatives depending on its first argument.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_IF(cond)(...)(...)
cond

The condition that determines whether the macro expands to the first or second __VA_ARGS__. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

[...]

The resulting expression if bit is non-zero. This is the first of the two __VA_ARGS__ in the syntax.

[...]

The resulting expression if bit is 0. This is the second of the two __VA_ARGS__ in the syntax.

Remarks

If cond expands to 0, this macro expands to the second __VA_ARGS__. Otherwise, it expands to the first __VA_ARGS__.

This macro performs a Boolean conversion on cond. If this conversion is undesirable, CHAOS_PP_VARIADIC_IIF may be used instead.

This macro is lazy. If cond is non-zero, only the first __VA_ARGS__ is evaluated. Likewise, if bit is 0, only the second __VA_ARGS__ is evaluated.

This macro is similar to CHAOS_PP_IF except that it allows variadic content in its true branch at the cost of preventing expansion of a function-like macros resulting from that branch. 

#define A() 1
#define B() 2

CHAOS_PP_VARIADIC_IF(1)(A, B)() // A()
CHAOS_PP_VARIADIC_IF(0)(A, B)() // 2

CHAOS_PP_VARIADIC_IF_ID

The CHAOS_PP_VARIADIC_IF_ID macro indirectly expands to CHAOS_PP_VARIADIC_IF.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_IF_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_VARIADIC_IF_

The CHAOS_PP_VARIADIC_IF_ macro expands to a λ expression representing CHAOS_PP_VARIADIC_IF.

Usage - C99 Specific 

CHAOS_PP_VARIADIC_IF_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_VARIADIC_IF)
