The repetition/enum_shifted.h header defines an algorithm that enumerates a user-specified target a specified number of times.

Usage

#include <chaos/preprocessor/repetition/enum_shifted.h>

Contents

  • CHAOS_PP_ENUM_SHIFTED

CHAOS_PP_ENUM_SHIFTED

The CHAOS_PP_ENUM_SHIFTED macro enumerates its second argument one less time than the number specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_SHIFTED(count, macro))
count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_SHIFTED(count, macro, ...))
count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_ENUM_SHIFTED(count, macro, data))
count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data)
data

Auxiliary data to be passed through the algorithm to the target.

Remarks

The CHAOS_PP_ENUM_SHIFTED macro itself expands to a deferred expression in terms of CHAOS_PP_NEXT(s). This deferred expression, when invoked by CHAOS_PP_EXPR, expands to the result of enumerating the target with a comma count - 1 times. This is illustrated by the following pseudo-code: 

macro(s, 1, data), macro(s, 2, data), ... macro(s, count - 1, data)

The target is invoked repeatedly (from 1 to count - 1) with the current recursion state (s), the current repetition (n), and, if it exists, the auxiliary data (data or __VA_ARGS__). This target may expand to a deferred expression in terms of CHAOS_PP_NEXT(s). This allows the target to recurse on itself if necessary.

In C99, it is possible to omit the auxiliary data argument altogether. This is different than passing "nothing" as the auxiliary data. 

CHAOS_PP_EXPR(
    CHAOS_PP_ENUM_SHIFTED(10, MACRO,) // #1
    CHAOS_PP_ENUM_SHIFTED(10, MACRO)  // #2
)

The first line passes "nothing" as the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0,)

The second line omits the auxiliary data, which causes MACRO to be invoked as (e.g.)... 

MACRO(s, 0)

The fundamental difference is that, in the first line, the auxiliary data is empty (which might have a valid meaning as data), but, in the second line, there is no auxiliary data. Hence, the algorithm is overloaded on a true-optional parameter.

Invocations of the target are trampolined such that each invocation is invoked with CHAOS_PP_NEXT(s), where s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()). This maximizes the utility of the recursion backend.

Ignoring the complexity of macro (which is invoked exactly count - 1 times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n), where n is count.

This macro differs from CHAOS_PP_ENUM simply by omitting the first repetition which can be useful in many contexts.

This macro deduces the current recursion state and invokes CHAOS_PP_ENUM_SHIFTED_S. If the current recursion state is already known, direct use of CHAOS_PP_ENUM_SHIFTED_S is more efficient.

This macro may not be used if bypass semantics are in effect.

Sample Code

#include <chaos/preprocessor/arithmetic/inc.h>
#include <chaos/preprocessor/cat.h>
#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/lambda/ops.h>
#include <chaos/preprocessor/recursion/expr.h>
#include <chaos/preprocessor/repetition/enum.h>
#include <chaos/preprocessor/repetition/enum_shifted.h>

#define FIXED(s, n, text) text

#define TTP(s, n, id) \
    template< \
        CHAOS_PP_EXPR_S(s)(CHAOS_PP_ENUM_S( \
            s, CHAOS_PP_INC(n), FIXED, class \
        )) \
    > class id ## n \
    /**/

CHAOS_PP_EXPR(CHAOS_PP_ENUM_SHIFTED(3, TTP, T))
// template<class, class> class T1,
// template<class, class, class> class T2

#if CHAOS_PP_VARIADICS

??=include CHAOS_PP_PLACEHOLDERS(1)

CHAOS_PP_EXPR(CHAOS_PP_ENUM_SHIFTED(
    3,
    CHAOS_PP_LAMBDA(template<)
        CHAOS_PP_EXPR_S_(_0)(CHAOS_PP_ENUM_S_(
            _0, CHAOS_PP_INC_(_1), CHAOS_PP_LAMBDA_(class)
        ))
    CHAOS_PP_LAMBDA(>) CHAOS_PP_PRIMITIVE_CAT_(class T, _1)
))
// template<class, class> class T1,
// template<class, class, class> class T2

??=include CHAOS_PP_PLACEHOLDERS(0)

#endif

CHAOS_PP_ENUM_SHIFTED_ID

The CHAOS_PP_ENUM_SHIFTED_ID macro indirectly expands to CHAOS_PP_ENUM_SHIFTED.

Usage 

CHAOS_PP_ENUM_SHIFTED_ID()

Remarks

This macro effectively allows CHAOS_PP_ENUM_SHIFTED to pass through itself without becoming disabled.

CHAOS_PP_ENUM_SHIFTED_

The CHAOS_PP_ENUM_SHIFTED_ macro expands to a λ expression representing CHAOS_PP_ENUM_SHIFTED.

Usage - C99 Specific 

CHAOS_PP_ENUM_SHIFTED_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_ENUM_SHIFTED_ID)()
