The repetition/parametric/delineate_from_to.h header defines an algorithm that delineates a user-specified target for each number in a specified range which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/parametric/delineate_from_to.h>

Contents

CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC

The CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC macro delineates its fourth argument for each number in the range specified by its first two arguments.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC(from, to, sep, macro))
from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific

Remarks

This macro behaves identically to CHAOS_PP_DELINEATE_FROM_TO except that the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

Ignoring the complexity of macro (which is invoked exactly to - from times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(t-f+1), where t is to and f is from. These bounds preserve all low-order terms, which implies that the algorithm can repeat the target p∙(sω-s-1) times, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, and s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()).

This macro deduces the current recursion state and invokes CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_S. If the current recursion state is already known, direct use of CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_ID

The CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_ID macro indirectly expands to CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC.

Usage 

CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_ID()

Remarks

This macro effectively allows CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC to pass through itself without becoming disabled.

CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_

The CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_ macro expands to a λ expression representing CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC.

Usage - C99 Specific 

CHAOS_PP_DELINEATE_FROM_TO_PARAMETRIC_

Remarks

This macro is a shorthand form of the following λ binding:
