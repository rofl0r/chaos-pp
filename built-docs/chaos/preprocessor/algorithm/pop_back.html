The algorithm/pop_back.h header defines an algorithm that removes the last element from a generic data structure.

Usage

#include <chaos/preprocessor/algorithm/pop_back.h>

Contents

  • CHAOS_PP_POP_BACK

CHAOS_PP_POP_BACK

The CHAOS_PP_POP_BACK macro removes the last element from its argument.

Usage 

CHAOS_PP_POP_BACK(g)
g

The non-nil generic data structure whose last element is to be removed.

Remarks

This macro expands to a generic data structure of the same type as g that contains all of the elements of g except the last.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n), where n is the number of elements in g.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_POP_BACK_BYPASS must be used instead.

Sample Code

#include <chaos/preprocessor/algorithm/pop_back.h>
#include <chaos/preprocessor/list/core.h>

CHAOS_PP_POP_BACK((CHAOS_PP_LIST) (a, (b, (c, ...))))
// (CHAOS_PP_LIST) (a, (b, ...))

CHAOS_PP_POP_BACK_ID

The CHAOS_PP_POP_BACK_ID macro indirectly expands to CHAOS_PP_POP_BACK.

Usage 

CHAOS_PP_POP_BACK_ID()

Remarks

This macro effectively allows CHAOS_PP_POP_BACK to pass through itself without becoming disabled.

CHAOS_PP_POP_BACK_

The CHAOS_PP_POP_BACK_ macro expands to a λ expression representing CHAOS_PP_POP_BACK.

Usage - C99 Specific 

CHAOS_PP_POP_BACK_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_POP_BACK_ID)()

CHAOS_PP_POP_BACK_BYPASS

The CHAOS_PP_POP_BACK_BYPASS macro behaves identically to CHAOS_PP_POP_BACK except that it is parametized by the bypass recursion state.

Usage 

CHAOS_PP_POP_BACK_BYPASS(s, g)
s

The current bypass recursion state.

g

The non-nil generic data structure whose last element is to be removed.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_POP_BACK.

CHAOS_PP_POP_BACK_BYPASS_ID

The CHAOS_PP_POP_BACK_BYPASS_ID macro indirectly expands to CHAOS_PP_POP_BACK_BYPASS.

Usage 

CHAOS_PP_POP_BACK_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_POP_BACK_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_POP_BACK_BYPASS_

The CHAOS_PP_POP_BACK_BYPASS_ macro expands to a λ expression representing CHAOS_PP_POP_BACK_BYPASS.

Usage - C99 Specific 

CHAOS_PP_POP_BACK_BYPASS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_POP_BACK_BYPASS_ID)()
