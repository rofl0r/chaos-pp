The tuple/insert.h header defines an algorithm that inserts an element into a tuple

Usage

#include <chaos/preprocessor/tuple/insert.h>

Contents

CHAOS_PP_TUPLE_INSERT

The CHAOS_PP_TUPLE_INSERT macro inserts its third argument into its second argument before the index specified by its first argument.

Usage - C99 Specific 

CHAOS_PP_TUPLE_INSERT(i, tuple, x)
i

The position at which the element is to be inserted. Valid values range from 0 to CHAOS_PP_TUPLE_SIZE(tuple).

[tuple]

The tuple into which the element is to be inserted.

[x]

The element to be inserted.

Remarks

This macro expands to a tuple with the elements of tuple with x inserted at position i.

The efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ, where n is i.

This macro operates on the nil tuple representation used by the core tuple operations defined in <chaos/preprocessor/tuple/core.h>. Specifically, () is a one-element tuple, not a zero-element tuple.

This macro uses bypass semantics. If bypass semantics are already in effect, CHAOS_PP_TUPLE_INSERT_BYPASS must be used instead.

See Also

  • CHAOS_PP_INSERT
  • CHAOS_PP_SEQ_INSERT

Sample Code

#include <chaos/preprocessor/config.h>
#include <chaos/preprocessor/tuple/insert.h>

#if CHAOS_PP_VARIADICS

CHAOS_PP_TUPLE_INSERT(0, (a, b, c), _) // (_, a, b, c)
CHAOS_PP_TUPLE_INSERT(1, (a, b, c), _) // (a, _, b, c)
CHAOS_PP_TUPLE_INSERT(2, (a, b, c), _) // (a, b, _, c)
CHAOS_PP_TUPLE_INSERT(3, (a, b, c), _) // (a, b, c, _)

#endif

CHAOS_PP_TUPLE_INSERT_ID

The CHAOS_PP_TUPLE_INSERT_ID macro indirectly expands to CHAOS_PP_TUPLE_INSERT.

Usage - C99 Specific 

CHAOS_PP_TUPLE_INSERT_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_INSERT to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_INSERT_

The CHAOS_PP_TUPLE_INSERT_ macro expands to a λ expression representing CHAOS_PP_TUPLE_INSERT.

Usage - C99 Specific 

CHAOS_PP_TUPLE_INSERT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_TUPLE_INSERT_ID)()

CHAOS_PP_TUPLE_INSERT_BYPASS

The CHAOS_PP_TUPLE_INSERT_BYPASS macro behaves identically to CHAOS_PP_TUPLE_INSERT except that it is parametized by the bypass recursion state.

Usage - C99 Specific 

CHAOS_PP_TUPLE_INSERT_BYPASS(s, i, tuple, x)
s

The current bypass recursion state.

i

The position at which the element is to be inserted. Valid values range from 0 to CHAOS_PP_TUPLE_SIZE(tuple).

[tuple]

The tuple into which the element is to be inserted.

[x]

The element to be inserted.

Remarks

If bypass semantics are already in effect, this macro must be used instead of CHAOS_PP_TUPLE_INSERT.

CHAOS_PP_TUPLE_INSERT_BYPASS_ID

The CHAOS_PP_TUPLE_INSERT_BYPASS_ID macro indirectly expands to CHAOS_PP_TUPLE_INSERT_BYPASS.

Usage - C99 Specific 

CHAOS_PP_TUPLE_INSERT_BYPASS_ID()

Remarks

This macro effectively allows CHAOS_PP_TUPLE_INSERT_BYPASS to pass through itself without becoming disabled.

CHAOS_PP_TUPLE_INSERT_BYPASS_

The CHAOS_PP_TUPLE_INSERT_BYPASS_ macro expands to a λ expression representing CHAOS_PP_TUPLE_INSERT_BYPASS.

Usage - C99 Specific 

CHAOS_PP_TUPLE_INSERT_BYPASS_
