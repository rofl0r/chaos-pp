The seq/concat.h header defines an algorithm that concatenates the elements of a sequence.

Usage

#include <chaos/preprocessor/seq/concat.h>

Contents

  • CHAOS_PP_SEQ_CONCAT

CHAOS_PP_SEQ_CONCAT

The CHAOS_PP_SEQ_CONCAT macro concatenates the elements of its argument.

Usage - C99 Specific 

CHAOS_PP_SEQ_CONCAT(seq)
[seq]

The sequence whose elements are to be concatenated.

Usage - C/C++ Specific 

CHAOS_PP_SEQ_CONCAT(seq)
seq

The sequence whose elements are to be concatenated.

Remarks

This macro performs an ordered concatenation of the elements of seq beginning with the first element. This is illustrated by the following pseudo-code: 

( ... ((seq[0] ## seq[1]) ## seq[2]) ... ## seq[length(seq) - 1])

In C99, if seq contains non-unary elements, the sub-elements of each element will also be concatenated. For example, 

CHAOS_PP_SEQ_CONCAT((a, b)(c, d)) // abcd

CHAOS_PP_SEQ_CONCAT_ID

The CHAOS_PP_SEQ_CONCAT_ID macro indirectly expands to CHAOS_PP_SEQ_CONCAT.

Usage 

CHAOS_PP_SEQ_CONCAT_ID()

Remarks

This macro effectively allows CHAOS_PP_SEQ_CONCAT to pass through itself without becoming disabled.

CHAOS_PP_SEQ_CONCAT_

The CHAOS_PP_SEQ_CONCAT_ macro expands to a λ expression representing CHAOS_PP_SEQ_CONCAT.

Usage - C99 Specific 

CHAOS_PP_SEQ_CONCAT_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_SEQ_CONCAT_ID)()
