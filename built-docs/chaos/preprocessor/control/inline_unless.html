The control/inline_unless.h header defines a macro that expands to an expression if a condition is 0.

Usage

#include <chaos/preprocessor/control/inline_unless.h>

Contents

  • CHAOS_PP_INLINE_UNLESS

CHAOS_PP_INLINE_UNLESS

The CHAOS_PP_INLINE_UNLESS macro expands to its second argument if its first argument is 0.

Usage - C99 Specific 

CHAOS_PP_INLINE_UNLESS(bit)(...)
bit

The condition that determines whether the macro should expand to __VA_ARGS__ or nothing. Valid values are 0 and 1.

[...]

The resulting expression if bit is 0.

Usage - C/C++ Specific 

CHAOS_PP_INLINE_UNLESS(bit)(x)
bit

The condition that determines whether the macro should expand to x or nothing. Valid values are 0 and 1.

x

The resulting expression if bit is 0.

Remarks

If bit expands to 1, this macro expands to nothing. Otherwise, it expands to x or __VA_ARGS__.

This macro does not perform a Boolean conversion on bit. If this conversion is desirable, CHAOS_PP_UNLESS may be used instead.

This macro is lazy. If bit is 1, x (or __VA_ARGS__) is not evaluated.

Sample Code

#include <chaos/preprocessor/control/inline_unless.h>

#define VALID() 1
#define INVALID() VALID(!) // too many arguments

CHAOS_PP_INLINE_UNLESS(1)(INVALID()) //
CHAOS_PP_INLINE_UNLESS(0)(VALID())   // 1

CHAOS_PP_INLINE_UNLESS_ID

The CHAOS_PP_INLINE_UNLESS_ID macro indirectly expands to CHAOS_PP_INLINE_UNLESS.

Usage 

CHAOS_PP_INLINE_UNLESS_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_INLINE_UNLESS_

The CHAOS_PP_INLINE_UNLESS_ macro expands to a λ expression representing CHAOS_PP_INLINE_UNLESS.

Usage - C99 Specific 

CHAOS_PP_INLINE_UNLESS_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_INLINE_UNLESS)
