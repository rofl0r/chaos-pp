The repetition/x/delineate_shifted_trailing.h header defines an exponential algorithm that delineates a user-specified target a specified number of times which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/x/delineate_shifted_trailing.h>

Contents

  • CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X

The CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X macro delineates its fourth argument the number of times specified by its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X(size, count, sep, macro))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

The delineator to be situated between each repetition of macro. This delineator must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

sep()
macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X(size, count, sep, macro, ...))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

count

One greater than the number of times that macro is to be repeated. Valid values range from 0 to CHAOS_PP_LIMIT_MAG.

sep

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n, data†)

† The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

[...]

Auxiliary data to be passed through the algorithm to the target.

data

Remarks

This macro behaves identically to CHAOS_PP_DELINEATE_SHIFTED_X except that it inserts a leading delineator if any repetitions exist.

This macro deduces the current recursion state and invokes CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X_S. If the current recursion state is already known, direct use of CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X_ID

The CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X_ID macro indirectly expands to CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X.

Usage 

CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X_ID()

Remarks

This macro effectively allows CHAOS_PP_DELINEATE_SHIFTED_TRAILING_X to pass through itself without becoming disabled.
