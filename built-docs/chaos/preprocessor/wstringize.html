The wstringize.h header defines wide-stringizing macros.

Usage

#include <chaos/preprocessor/wstringize.h>

Contents

  • CHAOS_PP_PRIMITIVE_WSTRINGIZE

CHAOS_PP_WSTRINGIZE

The CHAOS_PP_WSTRINGIZE macro wide-stringizes its argument after it has been expanded.

Usage - C99 Specific 

CHAOS_PP_WSTRINGIZE(...)
[...]

The argument to be wide-stringized.

Usage - C/C++ Specific 

CHAOS_PP_WSTRINGIZE(x)
x

The argument to be wide-stringized.

Remarks

This macro allows its argument to expand on input. This macro's sibling, CHAOS_PP_PRIMITIVE_WSTRINGIZE, does not provide this delay.

Both leading and trailing whitespace are removed from the argument when it is stringized, and internal whitespace separations are each condensed to a single space character. 

CHAOS_PP_WSTRINGIZE( abc ) // L"abc"
CHAOS_PP_WSTRINGIZE(1   2) // L"1 2"

In C99, if __VA_ARGS__ expands to a placemarker, this macro expands to an empty wide string literal. 

CHAOS_PP_WSTRINGIZE() // L""

Sample Code

#include <chaos/preprocessor/wstringize.h>

#define A text

CHAOS_PP_WSTRINGIZE(A) // L"text"

CHAOS_PP_WSTRINGIZE_ID

The CHAOS_PP_WSTRINGIZE_ID macro indirectly expands to CHAOS_PP_WSTRINGIZE.

Usage 

CHAOS_PP_WSTRINGIZE_ID()

Remarks

This macro exists only for uniformity with other macros that require indirect invocation.

CHAOS_PP_WSTRINGIZE_

The CHAOS_PP_WSTRINGIZE_ macro expands to a λ expression representing CHAOS_PP_WSTRINGIZE.

Usage - C99 Specific 

CHAOS_PP_WSTRINGIZE_

Remarks

This macro is a shorthand form of the following λ binding: 

CHAOS_PP_LAMBDA(CHAOS_PP_WSTRINGIZE)

CHAOS_PP_PRIMITIVE_WSTRINGIZE

The CHAOS_PP_PRIMITIVE_WSTRINGIZE macro wide-stringizes its argument before it is expanded.

Usage - C99 Specific 

CHAOS_PP_PRIMITIVE_WSTRINGIZE(...)
[...]

The argument to be wide-stringized.

Usage - C/C++ Specific 

CHAOS_PP_PRIMITIVE_WSTRINGIZE(x)
x

The argument to be wide-stringized.

Remarks

This macro behaves identically to CHAOS_PP_WSTRINGIZE except that it does not allow its argument to expand before it is wide-stringized.
