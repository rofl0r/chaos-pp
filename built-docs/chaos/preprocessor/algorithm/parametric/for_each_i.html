The algorithm/parametric/for_each_i.h header defines an algorithm that repeats a user-specified target for each element in a generic data structure which can be parametrically resumed.

Usage

#include <chaos/preprocessor/algorithm/parametric/for_each_i.h>

Contents

  • CHAOS_PP_FOR_EACH_I_PARAMETRIC

CHAOS_PP_FOR_EACH_I_PARAMETRIC

The CHAOS_PP_FOR_EACH_I_PARAMETRIC macro repeats the target specified by its first argument for each element in its second argument.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_I_PARAMETRIC(macro, g))
macro

The target that is to be repeated for each element in g. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem†)

† If the type of g is non-variadic, elem must be ... or the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be ... or a single parameter.

g

The generic data structure for which macro is to be repeated for each element.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_I_PARAMETRIC(macro, g, ...))
macro

The target that is to be repeated for each element in g. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem†, data‡)

† If the type of g is non-variadic, elem must be the number of parameters that corresponds to the number of sub-elements in each element of g. Otherwise, elem must be a single parameter.

‡ The data pseudo-parameter must be either ... or the number of parameters that corresponds to the number of arguments in the auxiliary data, __VA_ARGS__.

g

The generic data structure for which macro is to be repeated for each element.

[...]

Auxiliary data to be passed through the algorithm to the target.

Usage - C/C++ Specific 

CHAOS_PP_EXPR(CHAOS_PP_FOR_EACH_I_PARAMETRIC(macro, g, data))
macro

The target that is repeated for each element in g. This target must be a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, i, elem, data)
g

The generic data structure for which macro is to be repeated for each element.

data

Auxiliary data to be passed through the algorithm to the target.

Remarks

This macro behaves identically to CHAOS_PP_FOR_EACH_I except that the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

Ignoring the complexity of macro (which is invoked once for each element in g), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(n+1), where n is the number of elements in g. These bounds preserve all low-order terms, which implies that the algorithm can process exactly min(p∙(sω-s-1), m+1) elements, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the original recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), and m is CHAOS_PP_LIMIT_MAG.

This macro deduces the current recursion state and invokes CHAOS_PP_FOR_EACH_I_PARAMETRIC_S. If the current recursion state is already known, direct use of CHAOS_PP_FOR_EACH_I_PARAMETRIC_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_FOR_EACH_I_PARAMETRIC_ID

The CHAOS_PP_FOR_EACH_I_PARAMETRIC_ID macro indirectly expands to CHAOS_PP_FOR_EACH_I_PARAMETRIC.

Usage 

CHAOS_PP_FOR_EACH_I_PARAMETRIC_ID()

Remarks

This macro effectively allows CHAOS_PP_FOR_EACH_I_PARAMETRIC to pass through itself without becoming disabled.
