The repetition/x/repeat_from_to.h header defines an exponential algorithm that repeats a user-specified target for each number in a specified range which can be parametrically resumed.

Usage

#include <chaos/preprocessor/repetition/x/repeat_from_to.h>

Contents

CHAOS_PP_REPEAT_FROM_TO_X

The CHAOS_PP_REPEAT_FROM_TO_X macro repeats its fourth argument for each number in the range specified by its second and third arguments.

Usage - C99 Specific 

CHAOS_PP_EXPR(CHAOS_PP_REPEAT_FROM_TO_X(size, from, to, macro))
size

The size of the buffer of valid recursion steps maintained by the algorithm.

from

The lower bound of the range of numbers for which macro is to be repeated. Valid values range from 0 to to.

to

The upper bound of the range of numbers for which macro is to be repeated. Valid values range from from to CHAOS_PP_LIMIT_MAG.

macro

The target that is to be repeated. This target must be either a λ expression or a macro name that refers to a macro with a signature compatible with the following pseudo-signature: 

macro(s, n)

Remarks

This macro behaves identically to CHAOS_PP_REPEAT_FROM_TO except that the invocations of the target are not trampolined, the algorithm makes exponential use of the recursion backend, and the algorithm itself yields a trampolined deferred expression whenever the number of recursion steps is exhausted. This allows for parametric resumption.

This algorithm maintains a buffer of size recursion steps available at all times. This guarantees that the target has at least size recursion steps available whenever it is invoked.

The exponential structure of the X-suffixed algorithms yields a total of α recursion steps, where p is the number of parametric resumptions plus the initial entry, sω is CHAOS_PP_LIMIT_EXPR, s is the current recursion state at the beginning of the algorithm (i.e. CHAOS_PP_STATE()), sδ is the buffer size, and z is the exponential base. Currently, z is fixed at two in the CHAOS_PP_REPEAT_FROM_TO_X algorithm (though it could easily be increased), which reduces the equation to β. Ignoring the complexity of macro (which is invoked exactly to - from times), the efficiency of the algorithm's use of the recursion backend can be described in terms of the number of recursion steps required to complete the algorithm. The asymptotic upper and lower bounds are Θ(t-f+1), where t is to and f is from. These bounds preserve all low-order terms, which implies that the algorithm can generate exactly γ repetitions, where m is CHAOS_PP_LIMIT_MAG.

This macro deduces the current recursion state and invokes CHAOS_PP_REPEAT_FROM_TO_X_S. If the current recursion state is already known, direct use of CHAOS_PP_REPEAT_FROM_TO_X_S is more efficient.

This macro may not be used if bypass semantics are in effect.

CHAOS_PP_REPEAT_FROM_TO_X_ID

The CHAOS_PP_REPEAT_FROM_TO_X_ID macro indirectly expands to CHAOS_PP_REPEAT_FROM_TO_X.

Usage 

CHAOS_PP_REPEAT_FROM_TO_X_ID()

Remarks

This macro effectively allows CHAOS_PP_REPEAT_FROM_TO_X to pass through itself without becoming disabled.
